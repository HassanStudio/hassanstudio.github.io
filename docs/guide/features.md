# 功能介绍

Hassan 提供了一系列强大而实用的功能，让你在开发过程中更加高效。本页面将详细介绍 Hassan 的所有功能模块。

## 脚本执行引擎

Hassan 的核心是一个强大的脚本执行引擎，支持多种脚本语言。

### 支持的脚本语言

- **Shell 脚本**: Bash、Zsh、Sh 等
- **Python**: Python 2 / Python 3
- **Ruby**: Ruby 脚本
- **Swift**: Swift 脚本
- **其他**: 任何有 shebang 行的可执行脚本

### 智能解释器匹配

Hassan 会自动分析脚本的 shebang 行（如 `#!/bin/bash`），智能选择合适的解释器来执行脚本。你不需要手动指定如何运行脚本。

### 文件路径传递

当脚本被执行时，Hassan 会将文件的绝对路径作为第一个参数传递给脚本：

```bash
#!/bin/bash
# $1 = 文件的绝对路径
echo "处理文件: $1"
```

## 任务管理系统

Hassan 提供了完善的任务管理系统，让你轻松组织和管理所有脚本。

### 任务目录

Hassan 会自动扫描以下目录中的脚本：

```
~/Library/Application Support/Hassan/Tasks/
```

首次启动时，Hassan 会自动创建这个目录，并复制内置脚本到该位置。

### 层级组织

你可以使用文件夹来组织脚本，Hassan 会以树形结构展示：

```
~/Library/Application Support/Hassan/Tasks/
├── copy-absolute-path.sh          # 内置：复制绝对路径
├── copy-relative-path.sh          # 内置：复制相对路径
├── open-in-finder.sh              # 内置：在 Finder 中显示
└── custom/                        # 自定义脚本目录
    ├── development/               # 开发相关脚本
    │   ├── format-code.sh
    │   └── run-tests.sh
    └── utilities/                 # 实用工具脚本
        ├── convert-format.py
        └── analyze-file.swift
```

### 脚本元数据

你可以在脚本中使用注释添加元数据，Hassan 会解析并显示：

```bash
#!/bin/bash
# @title 格式化代码
# @description 使用 SwiftFormat 格式化 Swift 代码
# @author Rakuyo

# 脚本内容...
```

支持的元数据标签：
- `@title`: 脚本的显示标题
- `@description`: 脚本的详细描述
- `@author`: 脚本作者

### 智能过滤

Hassan 会自动过滤以下文件，不显示在任务列表中：
- 隐藏文件（以 `.` 开头）
- 临时文件（`~` 结尾、`.tmp`、`.temp` 等）
- 系统文件（`.DS_Store` 等）

## 内置脚本

Hassan 提供了三个实用的内置脚本，开箱即用。

### 1. 复制绝对路径

将文件的完整绝对路径复制到剪贴板。

**使用场景**：
- 在终端中引用文件
- 在脚本中使用完整路径
- 与其他工具共享文件位置

**示例输出**：
```
/Users/username/Projects/MyApp/Sources/AppDelegate.swift
```

### 2. 复制相对路径

智能计算并复制文件相对于项目根目录的路径。

**项目根目录检测逻辑**（按优先级）：
1. `.xcworkspace` - Xcode Workspace（monorepo）
2. `Tuist/` + `mise.toml` - Tuist 项目
3. `.git` - Git 仓库根目录
4. `.xcodeproj` - Xcode 项目

**使用场景**：
- 在配置文件中引用其他文件
- 编写项目文档
- 配置构建脚本
- 团队协作时共享文件位置

**示例输出**：
```
Sources/AppDelegate.swift
```

::: tip 提示
相对路径脚本会从文件所在目录开始，向上查找项目根目录标识。这意味着它可以正确处理嵌套的项目结构。
:::

### 3. 在 Finder 中显示

在 Finder 中定位并选中文件。

**使用场景**：
- 快速定位文件位置
- 重命名或移动文件
- 查看文件所在文件夹的其他内容
- 通过 Finder 执行其他操作

## 文件关联管理

Hassan 提供了强大的文件关联管理功能，让你可以批量设置文件类型的默认打开方式。

### 批量设置关联

在设置窗口的「文件关联」标签页中：
1. 查看所有支持的文件扩展名
2. 勾选你想要关联到 Hassan 的文件类型
3. 点击「应用设置」
4. Hassan 会记住原始的关联设置

### 支持的文件类型

Hassan 支持常见的开发文件类型：
- **Swift**: `.swift`
- **Objective-C**: `.h`, `.m`, `.mm`
- **C/C++**: `.c`, `.cpp`, `.cc`, `.hpp`
- **配置文件**: `.json`, `.xml`, `.plist`, `.yaml`, `.yml`
- **脚本**: `.sh`, `.py`, `.rb`, `.swift`
- **其他**: `.txt`, `.md` 等

::: info 自定义文件类型
如果需要支持其他文件类型，请参考[自定义脚本](/guide/custom-scripts)章节。
:::

### 恢复默认设置

Hassan 会在应用文件关联时保存原始设置。你可以随时：
1. 打开设置窗口
2. 切换到「文件关联」标签页
3. 点击「恢复默认」按钮
4. 所有文件关联会恢复到 Hassan 关联之前的状态

## 快捷任务

快捷任务功能让你可以设置一个默认脚本，提升高频操作的效率。

### 配置快捷任务

1. 打开设置窗口（`Command + ,`）
2. 在「通用」标签页中找到「快捷任务」设置
3. 勾选「启用」复选框
4. 从下拉菜单中选择默认脚本
5. （可选）勾选「快捷模式」

### 快捷模式

启用快捷模式后：
- 通过「在外部编辑器中打开」时会直接执行默认脚本
- 不会显示任务选择菜单
- 执行速度更快

**适合场景**：
- 你有一个最常用的脚本（如复制相对路径）
- 希望减少操作步骤
- 需要快速执行固定任务

**不适合场景**：
- 经常需要执行不同的脚本
- 需要灵活选择任务

::: warning 注意
启用快捷模式后，如需执行其他脚本，请通过菜单栏的 Hassan 图标访问完整的任务列表。
:::

### 禁用快捷任务

如果你不需要快捷任务功能：
1. 打开设置窗口
2. 取消勾选「启用」复选框
3. 每次都会显示任务选择菜单

## 菜单栏集成

Hassan 提供了菜单栏图标，方便你快速访问所有功能。

### 菜单栏功能

点击菜单栏中的 Hassan 图标，你可以：

- **任务** - 查看并执行所有可用脚本
- **偏好设置...** - 打开设置窗口（`Command + ,`）
- **关于 Hassan** - 查看应用信息和版本号
- **退出 Hassan** - 退出应用（`Command + Q`）

### 显示/隐藏菜单栏图标

如果你不希望在菜单栏显示 Hassan 图标：
1. 打开设置窗口
2. 在「通用」标签页中找到「显示菜单栏图标」
3. 取消勾选该选项

::: warning 注意
隐藏菜单栏图标后，你只能通过 Xcode 的「在外部编辑器中打开」功能来使用 Hassan。要重新显示图标，需要在应用程序文件夹中重新启动 Hassan。
:::

## 解释器配置

Hassan 需要知道如何执行不同类型的脚本，解释器配置功能让你可以管理这些设置。

### 自动检测解释器

1. 打开设置窗口
2. 切换到「解释器」标签页
3. 点击「自动检测系统解释器」按钮
4. Hassan 会扫描系统中可用的解释器

常见的解释器路径：
- **Bash**: `/bin/bash`, `/usr/local/bin/bash`
- **Zsh**: `/bin/zsh`, `/usr/local/bin/zsh`
- **Python**: `/usr/bin/python3`, `/usr/local/bin/python3`, `/opt/homebrew/bin/python3`
- **Ruby**: `/usr/bin/ruby`, `/usr/local/bin/ruby`
- **Swift**: `/usr/bin/swift`

### 调整优先级

如果系统中安装了多个版本的解释器（如 Python 2 和 Python 3），你可以：
1. 拖动列表中的解释器来调整优先级
2. Hassan 会优先使用排在前面的解释器

### 自定义解释器

你也可以手动添加或修改解释器：
1. 配置解释器的搜索路径
2. 设置解释器的命令名
3. Hassan 会验证解释器是否可用

## 首选项设置

Hassan 提供了丰富的自定义选项，让你可以根据自己的偏好配置应用。

### 外观设置

**外观模式**：
- 浅色模式
- 深色模式
- 跟随系统（自动切换）

Hassan 的界面会根据你的选择调整颜色主题。

### 语言设置

**支持的语言**：
- 简体中文
- English
- 跟随系统

Hassan 的所有界面文本都已完全国际化。

### 行为设置

**后台运行模式**：
- 启用：关闭窗口时应用保持在后台运行
- 禁用：关闭窗口时退出应用

**显示菜单栏图标**：
- 启用：在系统菜单栏显示 Hassan 图标
- 禁用：隐藏菜单栏图标

## 性能特点

Hassan 的设计注重性能和效率：

- **快速启动** - Hassan 启动时间小于 0.1 秒
- **低内存占用** - 运行时内存占用小于 20MB
- **即时响应** - 脚本执行和路径计算几乎是即时完成
- **异步执行** - 脚本在后台执行，不会阻塞界面

## 隐私与安全

Hassan 非常重视你的隐私和数据安全：

- ✅ **本地处理** - 所有操作都在本地完成，不会上传任何数据
- ✅ **无数据收集** - 不收集任何使用统计或分析数据
- ✅ **无网络连接** - 不需要网络连接即可正常工作
- ✅ **沙盒隔离** - 遵循 macOS 沙盒机制，保护系统安全
- ✅ **权限透明** - 只请求必要的权限，并明确说明用途
